# AgroScan Pro Development Rules

## Mission

Build the best agricultural intelligence platform for Africa.

---

Priority 1: Precision Agriculture

Priority 2: Satellite Analysis

Priority 3: Drone Analysis

Priority 4: Soil Analysis

Priority 5: AI Recommendations

Priority 6: Farm Management

Priority 7: WhatsApp Assistant

Priority 8: Reporting

Priority 9: Multi-language

Priority 10: Offline Support

---

## Quality Requirements

Every feature must be:

* Fast
* Accurate
* Scalable
* Secure
* Mobile First
* Production Ready

---

## Development Rules

* Precision Agriculture is the core of AgroScan.
* Satellite Analysis must be integrated into Crop Health.
* Drone Analysis must complement Satellite Analysis.
* Soil Analysis must enrich recommendations.
* AI must generate actionable recommendations.
* Reuse existing modules whenever possible.
* Do not create duplicate functionality.
* Prefer integration over new modules.
* Create tests for every new feature.
* Update documentation after implementation.
* Optimize for farmers first.
* Design for scalability from day one.

---

## Architecture Principles

One Platform.

One Database.

One Source of Truth.

Keep architecture simple.

Keep user experience simple.

Farmer First.

---

## AgroScan Vision

AgroScan must become the digital operating system for African agriculture.

Main users:

* Farmers
* Agricultural Advisors
* Cooperatives
* Agribusinesses
* NGOs
* Governments

Every feature must create measurable value for agricultural production.

## Technical Rules

Before coding:

1. Analyze existing code
2. Reuse existing architecture
3. Create implementation plan
4. Implement
5. Run tests
6. Produce report

Never create unnecessary modules.

Prefer extending existing modules.

Always keep PostgreSQL compatibility.

Always maintain API consistency.

Always maintain backward compatibility.
